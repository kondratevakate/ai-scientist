# Third-Party Sources And Redistribution Boundaries

This project studies third-party papers, repositories, databases, benchmarks,
blogs, and product pages. The public-domain dedication for this repository does
not relicense those sources.

## Local Snapshots

Local source snapshots are stored under:

`D:\projects\02_academia\ai-scientist\open-source`

These snapshots are for private inspection and reproducibility notes. They
should not be included in a clean public-domain release. Use `sources/code-repos.md` to
publish repository names, URLs, and commit provenance instead.

## Raw Search Exports

Raw search exports are stored under:

`D:\projects\02_academia\ai-scientist\searches`

These files may include third-party abstracts, metadata, database response
formats, API terms, or other content that is not owned by this project. They are
excluded from the default public release. The public compilation should publish
query logs, counts, screening decisions, and normalized source metadata instead.

## Citation And Summary Practice

Use short source-traceable summaries and stable links. Avoid copying paper
abstracts, long article passages, repository documentation, or website text into
the public compilation unless redistribution rights are clear.

## Third-Party License Audit Fields

For each source or repository, record where possible:

- source URL
- DOI / arXiv / canonical identifier
- upstream repository URL
- commit hash or release tag
- upstream license
- access date
- whether redistribution is allowed
- whether only a summary/link should be retained
